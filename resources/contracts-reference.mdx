This page provides an overview of all 35 contracts in the FHEVM Bootcamp curriculum.

## Contracts by Module

### Phase 1: Foundation (Modules 00-03)

<AccordionGroup>
  <Accordion title="01. SimpleStorage (Module 00)" icon="cube">
    **Purpose:** Solidity fundamentals review
    
    **Features:**
    - Basic storage variable
    - Getter and setter functions
    - Events
    
    **Key Learning:** Standard Solidity patterns before FHE
    
    **Deployed:** `0x8B7D25a45890d214db56790ae59afaE72273c1D3` (Sepolia)
  </Accordion>

  <Accordion title="02. BasicToken (Module 00)" icon="coins">
    **Purpose:** ERC-20 fundamentals review
    
    **Features:**
    - Plaintext balances
    - Transfer function
    - Mint function
    
    **Key Learning:** Understanding standard tokens before building confidential ones
    
    **Deployed:** `0x790f57EA01ec1f903645723D6990Eeaa2a36a814` (Sepolia)
  </Accordion>

  <Accordion title="03. HelloFHEVM (Module 02)" icon="hand-wave">
    **Purpose:** First encrypted contract
    
    **Features:**
    - Encrypted counter (`euint32`)
    - Increment function
    - Inherits `ZamaEthereumConfig`
    
    **Key Learning:** Basic encrypted state and FHE.allowThis()
    
    **Deployed:** `0xbFd008661B7222Dd974074f986D1eb18dD4dF1F1` (Sepolia)
  </Accordion>

  <Accordion title="04. EncryptedTypes (Module 03)" icon="shapes">
    **Purpose:** Type system demonstration
    
    **Features:**
    - All encrypted types (ebool, euint8-256, eaddress)
    - Type storage patterns
    - Type retrieval
    
    **Key Learning:** Encrypted type declaration and storage
    
    **Deployed:** `0x56c52A3b621346DC47B7B2A4bE0230721EE48c12` (Sepolia)
  </Accordion>

  <Accordion title="05. TypeConversions (Module 03)" icon="arrows-rotate">
    **Purpose:** Type casting demonstration
    
    **Features:**
    - Upcast (euint8 → euint32)
    - Downcast (euint32 → euint8)
    - Plaintext to encrypted conversion
    
    **Key Learning:** FHE.asEuintXX() for type conversion
    
    **Deployed:** `0x11c8ebc9A95B2A1DA4155b167dadA9B5925dde8f` (Sepolia)
  </Accordion>
</AccordionGroup>

### Phase 2: Core (Modules 04-09)

<AccordionGroup>
  <Accordion title="06. ArithmeticOps (Module 04)" icon="calculator">
    **Purpose:** Arithmetic operations on encrypted values
    
    **Features:**

    - FHE.div, FHE.rem (plaintext divisor)
    - FHE.min, FHE.max, FHE.neg
    
    **Key Learning:** All arithmetic FHE operations
    
    **Deployed:** `0xB6D81352EA3Cd0426838B655D15097E0FaE80177` (Sepolia)
  </Accordion>

  <Accordion title="07. BitwiseOps (Module 04)" icon="binary">
    **Purpose:** Bitwise operations
    
    **Features:**
    - FHE.and, FHE.or, FHE.xor, FHE.not
    - FHE.shl, FHE.shr (shift)
    - FHE.rotl, FHE.rotr (rotate)
    
    **Key Learning:** Bitwise manipulation of encrypted values
    
    **Deployed:** `0xb0bd1D30eDfaAbA1fc02F7A917820fD9edB24438` (Sepolia)
  </Accordion>

  <Accordion title="08. ComparisonOps (Module 04)" icon="code-compare">
    **Purpose:** Comparison operations
    
    **Features:**
    - FHE.eq, FHE.ne
    - FHE.gt, FHE.ge, FHE.lt, FHE.le
    - Returns ebool
    
    **Key Learning:** Encrypted comparisons produce ebool
    
    **Deployed:** `0xB1141F0b2588aAb0C1fe819b1B6AF1C0a7564490` (Sepolia)
  </Accordion>

  <Accordion title="09. ACLDemo (Module 05)" icon="lock">
    **Purpose:** Access Control List demonstration
    
    **Features:**
    - FHE.allowThis()

    - FHE.isSenderAllowed()
    
    **Key Learning:** ACL permission management
    
    **Deployed:** `0xc4f08eB557DF912E3D1FdE79bf3465d5242ea53d` (Sepolia)
  </Accordion>

  <Accordion title="10. MultiUserVault (Module 05)" icon="vault">

    **Features:**
    - mapping(address => euint64) balances
    - Deposit and withdraw
    - Per-user ACL
    
    **Key Learning:** Multi-user encrypted state management
    
    **Deployed:** `0xa988F5BFD7Fc19481Fdac5b55027b7Dc126a67e6` (Sepolia)
  </Accordion>

  <Accordion title="11. SecureInput (Module 06)" icon="upload">

    **Features:**
    - externalEuint32 input parameter
    - FHE.fromExternal(value, proof)
    - FHE.isInitialized() check
    
    **Key Learning:** Client-to-contract encrypted input pipeline
    
    **Deployed:** `0x27d2b5247949606f913Db8c314EABB917fcffd96` (Sepolia)
  </Accordion>

  <Accordion title="12. PublicDecrypt (Module 07)" icon="eye">
    **Purpose:** On-chain decryption pattern
    
    **Features:**
    - FHE.makePubliclyDecryptable()
    - FHE.isPubliclyDecryptable()
    - Timed reveal
    
    **Key Learning:** Public decryption for on-chain reveal
    
    **Deployed:** `0x605002BbB689457101104e8Ee3C76a8d5D23e5c8` (Sepolia)
  </Accordion>

  <Accordion title="13. UserDecrypt (Module 07)" icon="user-lock">
    **Purpose:** User-specific decryption via ACL
    
    **Features:**
    - FHE.allow(ciphertext, user)
    - Client-side instance.userDecrypt()
    - Private balance viewing
    
    **Key Learning:** Re-encryption for user-specific reads
    
    **Deployed:** `0x5E3ef9A91AD33270f84B32ACFF91068Eea44c5ee` (Sepolia)
  </Accordion>

  <Accordion title="14. ConditionalDemo (Module 08)" icon="code-branch">

    **Features:**
    - FHE.select(condition, valueIfTrue, valueIfFalse)
    - Nested selects
    - Boolean logic (FHE.and, FHE.or)
    
    **Key Learning:** Branchless conditional logic
    
    **Deployed:** `0x0A206f2BC275012703BA262B9577ABC49A4f6782` (Sepolia)
  </Accordion>

  <Accordion title="15. EncryptedMinMax (Module 08)" icon="arrows-up-down">

    **Features:**
    - FHE.gt comparison
    - FHE.select to choose min/max
    - Silent fail pattern
    
    **Key Learning:** Implementing encrypted min/max logic
    
    **Deployed:** `0xbA5c38093CefBbFA08577b08b0494D5c7738E4F6` (Sepolia)
  </Accordion>

  <Accordion title="16. RandomDemo (Module 09)" icon="dice">
